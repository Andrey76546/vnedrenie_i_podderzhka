## 🛠 Ход выполнения работы

### 1. Подготовка данных в Windows 7

* Создана папка «Для переноса» (For transferring), в которой размещен текстовый файл `Data.txt` с содержимым «Со старого ПК».


* Запущено «Средство переноса данных Windows» (путь: *Пуск > Все программы > Стандартные > Служебные > Средство переноса данных Windows*).


* В настройках выбора данных указан конкретный файл `Data.txt` для переноса.


* Сформирован файл переноса с расширением `.MIG` и сохранен в папку «Перенос» на рабочем столе.



### 2. Подготовка Windows 10

* Выполнена установка утилит виртуальной машины (VMware Tools) для обеспечения корректной работы с файловой системой.


* Папка «Перенос» с файлом `.MIG` скопирована из среды Windows 7 в среду Windows 10.


* Поскольку в Windows 10 штатное средство переноса отсутствует, использована утилита `migwiz` (скопирована из сетевого каталога `\\kbastrikin`).



### 3. Восстановление данных в Windows 10

* Запущена утилита `migwiz.exe`, выбран режим «Это мой новый компьютер».


* Указан путь к файлу переноса `.MIG` из папки «Перенос» на рабочем столе.


* После нажатия кнопки «Перенести» данные, включая файл `Data.txt`, были успешно восстановлены в соответствующий каталог.

## ❓ Ответы на контрольные вопросы

1. **Все ли файлы были успешно перенесены?**
Да, файл `Data.txt` был успешно перенесен и восстановлен в целевой директории на Windows 10, что подтверждается отчетом «Просмотр перенесенных элементов».


2. **Какие способы миграции (переноса) данных вы можете назвать?**
* Использование специализированного ПО для миграции (например, «Средство переноса данных Windows»).


* Ручное копирование данных через внешние носители (USB-флэш, внешние жесткие диски).


* Сетевой перенос (копирование файлов через локальную сеть или сетевые папки).


* Использование облачных сервисов хранения данных для синхронизации профиля пользователя.
