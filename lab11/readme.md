## 🛠 Ход выполнения работы

### 1. Чистая загрузка (Теоретическое обоснование)
В ходе работы был изучен механизм «чистой загрузки» (Selective Startup) через утилиту `msconfig`. Установлено, что данный режим позволяет загружать ОС только с базовым набором драйверов и служб, что необходимо для диагностики конфликтов стороннего программного обеспечения.

### 2. Проверка системных файлов (SFC)
Для обеспечения стабильности системы была проведена проверка целостности защищенных файлов Windows:
1. Запущена командная строка (CMD) от имени администратора.
2. Выполнена команда: `sfc /scannow`.
3. Результат: Система успешно просканирована, нарушений целостности не обнаружено.

### 3. Исследование параметров команды SFC
С помощью команды `sfc /help` были изучены ключи управления:
* `/scannow` — сканирование и автоматическое восстановление файлов.
* `/scanfile` — проверка конкретного файла.
* `/verifyonly` — проверка без внесения изменений.

## ❓ Ответы на контрольные вопросы

1. **Почему команда Sfc не может проверить созданный Вами файл sfccommand.txt?**
   Утилита SFC предназначена исключительно для проверки защищенных системных файлов Windows, входящих в состав ОС. Пользовательские файлы не являются объектами её контроля.

2. **С какими параметрами можно использовать команду sfc?**
   Основные параметры: `/scannow`, `/scanfile`, `/verifyonly`, `/help`.

3. **В каких ситуациях нужно использовать «чистую загрузку»?**
   При диагностике зависаний системы, конфликтов драйверов или ошибок, вызываемых сторонними приложениями, установленными в ОС.
