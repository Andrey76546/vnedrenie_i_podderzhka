### Цель работы

---

### Ход выполнения работы

* **Запуск и структура редактора:**
* Редактор запускается вводом команды `gpedit.msc` после нажатия клавиши «Пуск».


* Окно редактора разделено на две части: левая панель содержит иерархическую структуру политик, а правая — список политик выбранной категории.


* Редактор включает два основных раздела: «Конфигурация компьютера» и «Конфигурация пользователя».


* Политики конфигурации компьютера применяются ко всем пользователям, а политики конфигурации пользователя применяются только к текущему пользователю после входа в систему.




* **Практические манипуляции:**
* Основная идея подхода заключается в открытии веток дерева для поиска нужной категории политики.


* Выбор категории в левой панели приводит к отображению политик в правой панели.


* Столбец «Параметр» отображает имя политики, а столбец «Состояние» — её текущее состояние.


* Для изменения значений параметров, например, функции «Диалог слежения за завершением работы», можно использовать редактор реестра, переходя по пути `HKLM\Software\Policies\Microsoft\Windows NT\Reliability`.

### Контрольные вопросы

* **Что такое групповые политики?**
Это механизм, предназначенный для изменения групповых политик на локальном ПК.


* **Категории объектов групповых политик:**
Объекты делятся на категории «Конфигурация компьютера» и «Конфигурация пользователя».


* **Разделы групповых политик:**
Редактор включает разделы, такие как «Конфигурация компьютера» и «Конфигурация пользователя», с подразделом «Административные шаблоны» и другими.


* **Запуск групповых политик:**
Запуск осуществляется через команду `gpedit.msc` после нажатия кнопки «Пуск».
