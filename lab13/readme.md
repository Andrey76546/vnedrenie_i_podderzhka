## 🎯 Цель работы

## 🛠 Ход выполнения работы
### 1. Настройка сетевых параметров
Для соединения двух компьютеров были выполнены следующие шаги:
 * В «Центре управления сетями и общим доступом» открыты свойства адаптера Ethernet.
 * В протоколе IPv4 заданы статические IP-адреса:
   * **Компьютер №1:** 192.168.0.1, маска 255.255.255.0.
   * **Компьютер №2:** 192.168.0.2, маска 255.255.255.0, шлюз 192.168.0.1.
### 2. Конфигурация сети и доступ
 * Для обоих ПК задана одинаковая рабочая группа WORKGROUP и уникальные имена (Comp1, Comp2).
 * В параметрах общего доступа включен сетевое обнаружение и доступ к файлам.
 * Настроено «Предоставление доступа» для папок, чтобы пользователи другого ПК могли просматривать ресурсы.
### 3. Проверка соединения
 * С помощью команды ping 192.168.0.1 (с соседнего ПК) проверена работоспособность сетевого соединения.
 * Результат: пакеты успешно переданы, соединение установлено.
## ❓ Ответы на контрольные вопросы
 1. **Что такое IP-адрес?**
   Это уникальный сетевой идентификатор (адрес) устройства, подключенного к сети.
 2. **Типы IP-адресов:**
   Статические (постоянные) и динамические (выдаются сервером по DHCP).
 3. **Отличия IPv4 от IPv6:**
   IPv4 имеет адресное пространство 32 бит, IPv6 — 128 бит, что значительно увеличивает количество возможных адресов и решает проблему их нехватки.
 4. **Какой результат выполнения команды ping?**
   Команда ping проверяет сетевое соединение между узлами; положительный результат означает получение ответов от целевого компьютера, подтверждающих доступность узла в сети.
