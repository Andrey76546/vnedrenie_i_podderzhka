## 🛠 Методика проведения диагностики
Для выполнения работы был использован программный комплекс **PassMark BurnInTest v11.0**. Выбор данного ПО обусловлен его способностью создавать синхронную нагрузку, имитирующую экстремальные условия эксплуатации.

**Алгоритм тестирования:**
1. **Подготовка:** Инициализация диагностического комплекса, проверка актуальности драйверов.
2. **Конфигурация нагрузки:** Выбор ключевых узлов: ЦП, ОЗУ, накопитель, графический адаптер, аудиосистема и сетевой интерфейс.
3. **Исполнение:** Запуск непрерывного цикла тестирования длительностью 8 минут (с учетом лимитов пробной версии).
4. **Мониторинг:** Наблюдение за показателями «Cycle» (количество циклов) и «Operations» (объем выполненных операций) для оценки производительности.

## ❓ Контрольные вопросы
1. **Для чего необходимы программы для диагностики ПК:** Для контроля конфигурации, мониторинга износа носителей информации и предотвращения потери данных путем раннего обнаружения сбоев.
2. **Назначение BurnInTest:** Имитация долгосрочной эксплуатации ПК под высокой нагрузкой для выявления скрытых дефектов аппаратных компонентов.
3. **Что такое тестирование:** Это процесс сопоставления фактического поведения системы с ожидаемыми функциональными и техническими характеристиками.
4. **Виды тестирования:**
   * *Функциональное:* проверка работоспособности узлов.
   * *Нагрузочное:* работа на предельных мощностях.
   * *Тестирование на стабильность:* длительное исполнение кода для проверки надежности.
